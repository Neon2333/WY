  ````cpp
  ## 一、帧格式定义与环形缓冲区设计
  
  为了演示，我们假设一种简单的**变长帧协议**：
  
  | 起始标志   | 长度字段           | 数据部分 | 校验字节                     |
  | ---------- | ------------------ | -------- | ---------------------------- |
  | 0xAA, 0xBB | 1字节（数据长度n） | n字节    | 异或校验（从起始到数据末尾） |
  
  - 完整帧 = 起始标志(2) + 长度(1) + 数据(n) + 校验(1) = n+4 字节
  - 解析时先找 `0xAA, 0xBB`，读长度，再等够 n+4 字节，校验通过即提取。
  
  ### 1. 环形缓冲区类（RingBuffer）
  
  ```cpp
  // ringbuffer.h
  #pragma once
  #include <QByteArray>
  #include <QMutex>
  
  class RingBuffer
  {
  public:
      explicit RingBuffer(int capacity = 4096);
      void write(const QByteArray &data);        // 写入原始数据
      QByteArray extractFrame();                 // 尝试提取一个完整帧
  
  private:
      QByteArray buffer;          // 底层存储
      int readPos;                // 读指针位置
      int writePos;               // 写指针位置
      int capacity;
      QMutex mutex;               // 保证线程安全（可选，如果只在串口线程调用则不需要）
  };
  ```
  
  ```cpp
  // ringbuffer.cpp
  #include "ringbuffer.h"
  #include <QDebug>
  
  RingBuffer::RingBuffer(int capacity)
      : capacity(capacity), readPos(0), writePos(0)
  {
      buffer.resize(capacity);
  }
  
  //缓存数据
  void RingBuffer::write(const QByteArray &data)
  {
      QMutexLocker locker(&mutex);
      for (char byte : data) {
          buffer[writePos] = byte;
          writePos = (writePos + 1) % capacity;
          // 如果写指针追上了读指针，说明缓冲区满了，丢弃最老的数据（读指针前移）
          if (writePos == readPos) {
              readPos = (readPos + 1) % capacity;
          }
      }
  }
  
  //提取RawFrame
  QByteArray RingBuffer::extractFrame()
  {
      QMutexLocker locker(&mutex);
      int avail = (writePos - readPos + capacity) % capacity;
      if (avail < 4) return QByteArray();          // 连最小帧头都不够
  
      // 1. 寻找帧头 0xAA, 0xBB
      int start = -1;
      int tmp = readPos;
      for (int i = 0; i < avail - 1; ++i) {
          if (buffer[tmp] == char(0xAA) && buffer[(tmp+1)%capacity] == char(0xBB)) {
              start = tmp;
              break;
          }
          tmp = (tmp + 1) % capacity;
      }
      if (start == -1) {
          // 没有找到帧头，丢弃所有数据（防止无效数据堆积）
          readPos = writePos;
          return QByteArray();
      }
  
      // 2. 从 start 开始读取长度字段（第3个字节）
      int lenPos = (start + 2) % capacity;
      quint8 dataLen = static_cast<quint8>(buffer[lenPos]);
      int totalFrameLen = 4 + dataLen;     // 头2 + 长度1 + 数据n + 校验1
  
      // 3. 检查环形缓冲区中从 start 开始是否有足够字节
      int bytesFromStart = (writePos - start + capacity) % capacity;
      if (bytesFromStart < totalFrameLen) {
          return QByteArray();             // 数据不够，等待更多数据
      }
  
      // 4. 拷贝整个帧
      QByteArray frame;
      frame.reserve(totalFrameLen);
      int pos = start;
      for (int i = 0; i < totalFrameLen; ++i) {
          frame.append(buffer[pos]);
          pos = (pos + 1) % capacity;
      }
  
      // 5. 可选：校验帧（异或校验）
      quint8 xorSum = 0;
      for (int i = 0; i < totalFrameLen - 1; ++i) {
          xorSum ^= static_cast<quint8>(frame[i]);
      }
      if (xorSum != static_cast<quint8>(frame[totalFrameLen-1])) {
          // 校验失败，丢弃该帧，移动读指针跳过帧头继续寻找下一帧
          readPos = (start + 1) % capacity;
          return QByteArray();
      }
  
      // 6. 校验通过，移动读指针到帧末尾
      readPos = (start + totalFrameLen) % capacity;
      return frame;
  }
  ```
  
  ## 二、线程安全队列（生产者-消费者共享队列）
  
  ```cpp
  // threadsafequeue.h
  #pragma once
  #include <QQueue>
  #include <QMutex>
  #include <QWaitCondition>
  
  template<typename T>
  class ThreadSafeQueue
  {
  public:
      void push(const T &value)
      {
          QMutexLocker locker(&mutex);
          queue.enqueue(value);
          cond.wakeOne();          // 通知等待的消费者
      }
  
      T pop()                     // 阻塞等待
      {
          QMutexLocker locker(&mutex);
          while (queue.isEmpty()) {
              cond.wait(&mutex);
          }
          return queue.dequeue();
      }
  
      bool tryPop(T &value)       // 非阻塞尝试
      {
          QMutexLocker locker(&mutex);
          if (queue.isEmpty()) return false;
          value = queue.dequeue();
          return true;
      }
  
  private:
      QQueue<T> queue;
      QMutex mutex;
      QWaitCondition cond;
  };
  ```

  ```cpp
  // serialreader.cpp
  #include "serialreader.h"
  #include <QThread>
  
  SerialReader::SerialReader(const QString &portName, QObject *parent)
      : QObject(parent), ringBuffer(8192), running(false)
  {
      serial = new QSerialPort(this);
      serial->setPortName(portName);
      serial->setBaudRate(QSerialPort::Baud115200);
      serial->setDataBits(QSerialPort::Data8);
      serial->setParity(QSerialPort::NoParity);
      serial->setStopBits(QSerialPort::OneStop);
      // 其他参数按需设置
  }
  
  SerialReader::~SerialReader()
  {
      stop();
  }
  
  void SerialReader::start()
  {
      if (running) return;
      if (!serial->open(QIODevice::ReadOnly)) {
          qWarning() << "Failed to open serial port:" << serial->errorString();
          return;
      }
      running = true;
      connect(serial, &QSerialPort::readyRead, this, &SerialReader::handleReadyRead);
  }
  
  void SerialReader::stop()
  {
      if (!running) return;
      running = false;
      disconnect(serial, &QSerialPort::readyRead, this, nullptr);
      serial->close();
  }
  
  void SerialReader::handleReadyRead()
  {
      QByteArray data = serial->readAll();
      ringBuffer.write(data);
  
      // 循环提取完整帧
      QByteArray frame;
      while (true) 
      {
          frame = ringBuffer.extractFrame();
          if (frame.isEmpty()) break;
          emit frameReady(frame);   //发射信号，让主线程将帧入队并提交线程池任务
      }
  }
  ```
  
  **注意**：为了不把 `frameQueue` 直接暴露给 `SerialReader` 依赖，建议让主线程连接 `frameReady` 信号，在槽函数中将帧推入共享队列并提交解析任务。这样 `SerialReader` 保持整洁。

      threadPool = QThreadPool::globalInstance();
      threadPool->setMaxThreadCount(4);

      connect(serialThread, &QThread::finished, reader, &SerialReader::stop);
      connect(serialThread, &QThread::finished, reader, &QObject::deleteLater);
      serialThread->start();
  }
  
  void MainWindow::onFrameReady(const QByteArray &frame)
  {
      // 将帧推入共享队列
      frameQueue.push(frame);
  
      // 立即向线程池提交一个解析任务（每个帧一个任务）
      ParseTask *task = new ParseTask(&frameQueue, this);
      threadPool->start(task);
  }

  ## 六、关键点解释
  
  ### ✅ 为什么每个帧单独一个 ParseTask？
  
  - **简单**：每个任务独立，不会因为一个任务阻塞而影响后续帧。
  - **自动并行**：线程池根据可用线程数并发执行多个解析任务。
  - **注意**：如果帧产生速度远大于解析速度，队列会堆积。可以通过增加线程数、优化解析逻辑或改用**批量解析任务**来缓解。生产环境中，也可以让每个任务一次处理多个帧（循环 `tryPop` 直到空），减少任务提交开销。
  
  ### ✅ 环形缓冲区的作用
  
  - 解决了串口数据**粘包**（多个帧连在一起）和**拆包**（一个帧被分多次接收）问题。
  - 通过环形缓冲区保留未处理完的字节，每次 `extractFrame()` 尝试解析，能解析出完整帧就返回，否则等待更多数据。
  - 遇到无效数据（如没有找到帧头）时会丢弃，避免缓冲区被垃圾填满。
  
  ### ✅ 线程安全
  
  - `RingBuffer` 内部用互斥锁（本例中仅在串口线程使用，实际可省略）。
  - `ThreadSafeQueue` 用互斥锁+条件变量保证线程安全。
  - `ParseTask` 运行在线程池，通过 `frameQueue.pop()` 阻塞获取数据，无需自旋。
  
  ### ✅ 停止与清理
  
  - 主窗口析构时，需要停止串口线程并等待线程池结束：
  
    ```cpp
    ~MainWindow() {
        serialThread->quit();
        serialThread->wait();
        threadPool->waitForDone(5000);
    }
    ```
  
  - 注意：`ParseTask` 中 `pop()` 是阻塞的，如果程序退出时队列为空，任务会一直阻塞。解决办法：增加停止标志或使用超时 `tryPop` 或使用 `QWaitCondition::wait(&mutex, timeout)`。
  
  ## 七、进阶优化建议
  
  1. **批量处理**：让一个 `ParseTask` 在 `run()` 中循环 `tryPop` 处理多个帧，当队列为空时结束，减少任务创建开销。
  2. **解析结果回传**：使用自定义信号类，将解析结果通过 `QueuedConnection` 安全发送到主线程进行 UI 刷新。
  3. **流控**：当队列长度超过阈值时，通知生产者暂停读取，或丢弃低优先级帧。
  
  这套架构在多个工业级 Qt 串口设备（如医疗仪器、无人机遥测）中验证有效，希望对你有帮助。如果你对环形缓冲区的字节偏移处理或帧检验算法有疑问，欢迎继续探讨。
  ````

  ```sqlite
  -- 测试日志表
  CREATE TABLE log_test (
      id INTEGER PRIMARY KEY AUTOINCREMENT,   -- 自增主键
      device_id INTEGER NOT NULL,             -- 设备id，关联 para_device.id
      method_id INTEGER NOT NULL,             -- 方法id，关联 test_methods.id
      result REAL,                            -- 测试结果（double 映射为 REAL）
      FOREIGN KEY (device_id) REFERENCES para_device(id),
      FOREIGN KEY (method_id) REFERENCES test_methods(id)
  );
  
  -- 下发命令日志表
  CREATE TABLE log_cmd (
      id INTEGER PRIMARY KEY AUTOINCREMENT,   -- 自增主键
      cmd1 TEXT                               -- 下发命令参数1（类型未指定，暂用 TEXT）
  );
  ```
